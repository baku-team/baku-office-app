# Google 連携（サービスアカウント）の自動セットアップ

この手順は **Google Cloud Shell**（ブラウザ内のターミナル・ログイン済み）で実行します。
GCP プロジェクト作成・API有効化・サービスアカウント作成・鍵(JSON)発行までを自動で行い、
最後に baku-office の設定画面に貼る値を表示します。所要 2〜3 分です。

## 始める前に

- この Cloud Shell は、いま開いているあなたの Google アカウントとして動きます。
- プロジェクト作成や API 有効化の権限（と、必要なら課金アカウント）があるアカウントで開いてください。

最初に「承認（Authorize）」を求められたら **Authorize** を押してください。

## ステップ 1：セットアップを実行する

下のコマンドの実行ボタン（▶）を押すと、自動でセットアップが進みます。

```bash
bash google-service-account-setup.sh
```

途中で操作は不要です（API反映の待ちは自動でリトライします）。完了すると `[1/5]…[5/5]` の
進捗のあと、画面の最後に **鍵(JSON)** ・ **クライアントID** ・ **スコープ** がまとめて表示されます。

## ステップ 2：表示された鍵(JSON)をコピーする

ステップ1の出力の最後、**「┌─ ここから鍵(JSON)をコピー」と「└─ ここまで」の間**の JSON を
**丸ごとコピー**します（別コマンドは不要です）。

> ヒント：左の「エディタを開く」から `*-key.json` を開いてコピーしても、`cat *-key.json` を実行してもかまいません。

## ステップ 3：baku-office に貼り付けて登録する

baku-office の「Google連携セットアップ → サービスアカウント」画面で：

1. **鍵(JSON)** … ステップ2でコピーした内容を貼り付け（または鍵ファイルをアップロード）
2. **代理するユーザー** … 予定やメールを操作する Workspace ユーザーのメール
3. **利用する機能** を選んで「この内容で登録」

## ステップ 4：ドメイン全体の委任を承認する（1回だけ）

baku-office の画面に表示される **クライアントID** と **スコープ** を、
Google Workspace 管理コンソールの「ドメイン全体の委任」に登録します。

- 管理コンソール：<https://admin.google.com/ac/owl/domainwidedelegation>
- ここだけは Google がセキュリティ上、人による承認を必須にしているため自動化できません。

## 完了

承認後、baku-office の画面で「**接続テスト**」を押して成功すれば連携完了です。
このタブは閉じてかまいません。
