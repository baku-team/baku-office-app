# Google 連携（サービスアカウント）の自動セットアップ

この手順は **Google Cloud Shell**（ブラウザ内のターミナル・ログイン済み）で実行します。
GCP プロジェクト作成・API有効化・サービスアカウント作成・鍵(JSON)発行までを自動で行い、
最後に baku-office の設定画面に貼る値を表示します。所要 2〜3 分です。

## 始める前に

- この Cloud Shell は、いま開いているあなたの Google アカウントとして動きます。
- プロジェクト作成や API 有効化の権限（と、必要なら課金アカウント）があるアカウントで開いてください。

最初に「承認（Authorize）」を求められたら **Authorize** を押してください。

## ステップ 1：セットアップ（自動で始まります）

この画面を開くと、セットアップ用のコマンドが**自動で実行**されます
（「Authorize（承認）」や「コマンドを実行しますか？」の確認が出たら **許可**してください。
これは Google のセキュリティ上、省略できません）。

> 自動で始まらない／やり直す場合は、下の実行ボタン（▶）を押してください。
>
> ```bash
> bash google-service-account-setup.sh
> ```

最初に **代理ユーザー**（予定/メールを操作する Workspace ユーザー）の確認が表示されます。
ログイン中のアカウントでよければ **Enter**、別ユーザーならメールを入力してください。
以降は自動で進み（API反映の待ちは自動リトライ）、完了すると **鍵(JSON) は自動でクリップボードへ
コピー**され、画面の最後に **代理ユーザー・鍵・クライアントID・スコープ** が 1 画面にまとまります。

## ステップ 2：baku-office に貼り付けて「登録して接続テスト」

baku-office の「Google連携セットアップ → サービスアカウント」画面で：

1. **鍵(JSON)** … 入力欄をクリックして **そのまま貼り付け（Ctrl+V / Cmd+V）**。
   （クリップボードが効かない場合は、出力の「┌─ … └─」の間を手動コピー）
2. **代理するユーザー** … 既定で団体アカウントが入っています（変える場合のみ修正）
3. **利用する機能** を選んで **「登録して接続テスト」** を押す

## ステップ 3：ドメイン全体の委任を承認する（1回だけ）

「登録して接続テスト」の後に表示される **クライアントID** と **スコープ** を、
Google Workspace 管理コンソールの「ドメイン全体の委任」に登録します。

- 管理コンソール：<https://admin.google.com/ac/owl/domainwidedelegation>
- ここだけは Google がセキュリティ上、人による承認を必須にしているため自動化できません。

## 完了

承認したら、baku-office で **もう一度「登録して接続テスト」** を押してください。
成功すれば連携完了です（このタブは閉じてかまいません）。
